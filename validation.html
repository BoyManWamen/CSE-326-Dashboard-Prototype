Validation Rules

How we bucket each tube’s seed count per time slice.

Bucketing (current)

Seed count Bucket Meaning
0 skip Missed drop (gap)
1 ideal Correct single seed
2 double Two seeds (crowding / waste)
3+ overdrop Overdrop (severe)

In your Flask backend, this mapping lives in classify(seed_count). On GitHub Pages, the same logic is re-implemented in assets/app.js.

Defining “Valid Planting”

A simple validity rule for a time window could be:

valid = (ideal_rate ≥ threshold) AND (skip_rate ≤ max_skip) AND (overdrop_rate ≤ max_over)
  • ideal_rate = ideal / total observations
  • threshold might be 0.97 (97%) depending on stakeholder requirement
  • In production, compute these per tube and overall
Why this helps
It turns raw detection output into an actionable “pass/fail” signal the crew can use.
